第三次讲座

第 3次讲座：详细内容

军训第 3 周 · 80 分钟

主题：你的课表不是随机的

基调：把抽象的课程体系讲得有血有肉，开学后带着”地图感”去上课

讲前准备清单

准备事项 说明
学校的培养方案 下载完整版，打印或截图，现场展示真实课表
技能树大图 PPT 上做一张完整的四年课程技能树图，这是本次讲座最核心的视觉素材
数学应用场景的截图 线代用在 AI 里的截图、概率用在推荐系统里的截图等，让他们直观感受
上次作业检查 开场问一句”培养方案下载了吗？”，没下载的提醒一下

开场（5 分钟）

今天我们要做一件事——把你们接下来四年要上的课，从头到尾拆开看一遍。”

“先问一个问题：上周让你们下载培养方案，下载了吗？”

（停顿等反应）

“没下载的也别急，我今天会把重点的展示给你们看。但我还是建议你们回去下载一份——这是你们未来四年的’游戏攻略’，不看攻略就上路，很容易迷路。”

“今天我们不讲具体知识点，那些开学后老师会教。我想让你们理解的是：这些课为什么存在、它们之间有什么关系、学好它们意味着什么。“

核心概念：课程是一棵技能树（10 分钟）

“你们看到的课表，表面上是一堆独立的课程：高数、线代、C语言、大物……看起来毫无关联，像超市货架上的商品一样随机排列。”

“但真相是——它们是一棵技能树。 每门课都是树上的一环，有些是根，有些是枝，有些是叶。你不可能跳过根直接去摘叶子。”

（PPT 切到技能树全景图 或 白板手画）

画出这样的结构：

“我把它分成四层。从下往上，是你们从大一到大四的学习路径。从上往下，是每一层对下面的依赖关系。你学操作系统之前，必须先懂计算机组成原理；学组成原理之前，必须先会写 C 语言；学 C 语言之前，最好有一些离散数学的基础……”

“所有课程的排课顺序不是随便排的，是有逻辑链的。你大一翘的每一门课，大二大三都会找回来。”

“接下来我一层一层讲。”

第一层：数学地基（20 分钟）

“先说最底下这层——数学。我知道你们可能最不想听这个，但请给我 20 分钟，我保证讲完之后你不会再觉得数学’没用’。”

（PPT 切到数学四门课的页面）

高等数学

一句话： “研究变化的数学——极限、导数、积分、级数。”

生活中的例子：

“你开车，速度表显示 60km/h，这是’瞬时速度’。瞬时速度怎么算出来的？——对位移求导。这就是导数。你从 0 加速到 60km/h，走了多远？——对速度求积分。这就是积分。”

在 CS 中用在哪：

“机器学习训练模型，本质上就是找一个函数的最小值。怎么找？——梯度下降法。梯度是什么？——导数在多维空间的推广。所以你大三学机器学习的时候，底下撑着的就是大一学的微积分。”

“图形学里，光照怎么计算？——积分。物理引擎里，物体运动怎么模拟？——微分方程。”

给他们的建议：

“高数不需要你考 100 分，但至少要理解导数和积分的物理含义。不要只会套公式算题，要理解它在描述什么。”

线性代数

一句话： “研究向量和矩阵的数学——高维空间的语言。”

生活中的例子：

“你拍了一张照片。这张照片在计算机眼里是什么？——一个矩阵。每个像素点是一个数字，一张 1920×1080 的照片就是一个 1920×1080 的矩阵。你对照片做旋转、缩放、滤镜——本质都是矩阵运算。”

在 CS 中用在哪：

“这是跟 AI 关系最紧密的数学。 你们以后会听到一个词叫’神经网络’。神经网络在做什么？本质就是大量的矩阵乘法。一个大语言模型有几百亿个参数，这些参数怎么存的？——矩阵。怎么计算的？——矩阵乘法。”

“推荐系统给你推荐商品，本质上是把’你’和’商品’都表示成向量，然后算两个向量的距离。搜索引擎给你排序结果，也是向量运算。”

“Google 最早的 PageRank 算法——决定搜索结果排名的核心算法——本质上就是在解一个巨大的矩阵特征值问题。”

给他们的建议：

“线性代数是你们未来选择面最广的一门数学。认真学，特别是：向量、矩阵乘法、特征值和特征向量这三个概念。考试之后不要丢掉，大三学 AI 的时候你会感谢自己。”

概率论与数理统计

一句话： “研究不确定性和随机性的数学。”

生活中的例子：

“明天降水概率 70%——这是概率。你刷短视频的推荐为什么这么准？——背后是统计模型在算’你喜欢这个视频的概率有多大’。”

“一个经典问题：你参加一个节目，面前有三扇门，一扇后面有车，两扇后面是羊。你选了一扇门，主持人打开另一扇，后面是羊。他问你要不要换？——换不换？”

（停顿等反应）

“应该换。 换了之后赢的概率是 2/3。这就是著名的蒙提霍尔问题，反直觉但数学上可以严格证明。概率论会训练你用数学而非直觉去理解不确定性。”

在 CS 中用在哪：

“机器学习的根基就是概率统计。分类器在做什么？——算’这个样本属于某个类别的概率’。贝叶斯定理是垃圾邮件过滤器的基础。A/B 测试（决定用哪个版本的产品）靠的是统计推断。”

“甚至你打游戏，游戏里的随机暴击、掉宝率，底层都是概率模型。”

给他们的建议：

“概率论和高数不一样——高数偏计算，概率论偏思维。你学完之后对’随机性’的理解会完全改变。认真学贝叶斯定理，这一个定理就够你用一辈子。”

离散数学

一句话： “研究离散结构的数学——逻辑、集合、图、组合。这是最’计算机’的一门数学。”

生活中的例子：

“微信好友关系怎么表示？——图论里的’图’。每个人是一个节点，好友关系是一条边。六度分隔理论（你通过最多6个人就能认识世界上任何人）——这就是图论的问题。”

“一个快递员要送 10 个包裹，怎么规划路线最短？——这是组合优化问题。NP-hard，至今没有完美解法。”

在 CS 中用在哪：

“离散数学和 CS 的关系是最直接的：”

  • 逻辑 → 编译原理（代码怎么被解析的）、数据库查询优化

  • 集合论 → 数据库的理论基础（关系数据库就是建立在集合论上的）

  • 图论 → 网络路由、社交网络分析、推荐系统、地图导航

  • 组合数学 → 密码学、算法复杂度分析

  • 数理逻辑 → 人工智能推理、形式化验证

给他们的建议：

“离散数学可能是你们大一最’陌生’的一门数学——它不像高数那样有物理直觉可以依赖。但它跟 CS 的关系最紧密。认真学图论和逻辑，后面学数据结构、算法、数据库的时候会直接用到。“

数学部分小结

“我花这么多时间讲数学，是因为这是我最想让你们听到的。”

“你们大一学数学的时候，很可能会想：’我又不做研究，学这些干嘛？’我理解，因为我也这么想过。但事实是——“

“数学是你未来选择权的保障。“

“你现在不知道自己以后想做什么，很正常。但如果你大一把数学学好了，大三你想做 AI——可以，数学基础够；想做图形学——可以，线代够用；想考研——可以，数学分不会拖后腿。”

“反过来，如果你大一数学没学好，大三想做 AI，发现看不懂梯度下降、看不懂损失函数、看不懂论文里的公式——那时候补课的成本比现在高十倍。”

“你不需要考满分，但你需要真正理解。 不要为了考试而学，要为了’以后看得懂’而学。”

第二层：编程基础（15 分钟）

“数学讲完了，来看编程基础。这是你们大一到大二花时间最多的部分。”

C 语言程序设计

一句话： “你的第一门编程语言——也是最’底层’的一门。”

为什么要先学 C，而不是 Python？

“很多新生会问：’Python 更简单更流行，为什么不先学 Python？’原因很简单——C 语言让你理解计算机’真正的想法’。”

“Python 帮你做了很多事情：内存管理、类型转换……它把复杂的东西藏起来了，你用起来很舒服，但你不理解底下发生了什么。”

“C 语言逼你直面这些：指针是什么？内存怎么分配？变量存在哪里？这些概念痛苦，但它们帮你在脑子里建立一个’计算机是怎么工作的’的模型。以后不管你用什么语言，这个模型都会帮到你。”

打个比方：

“学 C 就像学手动挡开车。很痛苦，但学完之后你对车的理解比只学自动挡的人深得多。以后换什么车都能开。”

给他们的建议：

“C 语言的关键不是背语法，是理解内存、指针、地址这些概念。如果这三样搞懂了，后面换任何语言都很快。推荐书：《C Primer Plus》，厚但讲得清楚。”

数据结构

一句话： “用什么方式组织数据，决定了你的程序快不快。”

为什么要单独强调这门课？

“数据结构是 CS 最核心的课程，没有之一。 它是算法课的前置、是面试的重点、是以后写任何程序的基础。如果你整个大学只能学好一门课，我希望是这一门。”

用生活中的例子讲数据结构：

“假设你有一个书架，你要找一本书。”

  • 数组： 书从左到右排成一排，找某本书只能一本一本翻 → 简单但找起来慢

  • 链表： 每本书上贴着一张纸条写着’下一本在哪’ → 插入删除方便，但不能随机跳转

  • 栈： 像叠盘子，只能从最上面拿 → 后来放上去的先被拿走（后进先出）

  • 队列： 像排队，先来的先走 → 先进先出

  • 树： 像公司组织架构，CEO 下面是 VP，VP 下面是经理 → 层级结构，查找快

  • 哈希表： 像字典，你直接翻到’K’开头的部分找’Kangaroo’ → 瞬间定位

  • 图： 像地铁线路图，站点之间有多条连接 → 表示复杂的关系网络

“不同的数据结构适合不同的场景。选对了数据结构，程序快一万倍；选错了，再怎么优化都没用。”

给他们的建议：

“数据结构不要只看懂，要自己实现。链表自己写一遍、树自己建一棵、排序自己跑一遍。书推荐《大话数据结构》（通俗易懂），进阶看《算法导论》。另外，从大一开始刷 LeetCode 简单题，不为面试，为练手感。”

算法设计与分析

一句话： “数据结构解决’用什么方式存数据’，算法解决’用什么步骤处理数据’。”

为什么重要：

“同样的问题，不同的解法效率可能差百万倍。”

“举个例子：从 100 万个数字里找一个数。”

  • 暴力搜索：一个一个看 → 最坏看 100 万次

  • 二分查找（先排序好）→ 最多看 20 次

  • “100 万次 vs 20 次。差 5 万倍。这就是算法的力量。”

核心内容预告（让他们有个印象）：

“你们以后会学到这些：排序算法、搜索算法、动态规划、贪心算法、分治法……这些名字现在听不懂没关系，但记住：算法 = 解决问题的策略。同一道题，高手 3 行代码搞定，你写 100 行还超时——差别就在算法。”

给他们的建议：

“算法和数据结构是绑定的，一起学、一起练。LeetCode 从 Easy 开始刷，不要上来就做 Hard。ACM 竞赛感兴趣的话，大一下可以开始了解。”

面向对象程序设计（C++/Java）

一句话： “从写’小脚本’到写’大工程’的跨越。”

为什么重要：

“C 语言教你跟计算机对话，面向对象教你跟复杂度对话。”

“当你的程序只有 100 行，用 C 写没问题。但当你的程序有 10 万行、10 个人一起写——你需要一种方式来组织代码，让它不乱套。面向对象就是解决这个问题的。”

打个比方：

“如果 C 语言是教你做一道菜，面向对象就是教你开一个餐厅——厨师、服务员、采购、财务各管各的，但合在一起能运转。”

给他们的建议：

“重点理解三个概念：封装、继承、多态。不要死记语法，理解’为什么要这样设计’。Java 和 C++ 选一个深入学就行，不贪多。”

第三层：系统层（15 分钟）

“如果说数学和编程是地基和砖块，那系统层就是教你理解整栋建筑的结构。”

“这些课一般大二大三才上，但我现在告诉你们它们是什么，你们以后上到的时候就不会懵。”

计算机组成原理

一句话： “代码是怎么变成电信号的？”

“你写的代码，计算机其实看不懂。代码要经过编译变成机器指令，机器指令就是 0 和 1，0 和 1 在硬件层面就是高低电平。”

“组成原理就是教你理解：CPU 怎么执行一条指令？数据怎么在内存和 CPU 之间流动？硬盘怎么存数据？这些硬件是怎么协作的？”

打个比方：

“如果计算机是一辆车，组成原理就是教你理解发动机、变速箱、传动轴是怎么配合的。你不需要会造发动机，但你需要知道车是怎么跑起来的。”

给他们的建议：

“这门课偏硬件，有些同学会觉得枯燥。但它帮你建立’从硬件到软件’的完整认知。耐心学，特别是 CPU 流水线和存储层级这两块。”

操作系统

一句话： “一台电脑怎么同时干这么多事的？”

“你的电脑同时跑着十几个程序，每个程序都以为自己独占了整台电脑。但实际上，CPU 在不同程序之间飞速切换——每秒切换几百万次，快到你完全感觉不到。”

“操作系统就是这个’大管家’：管理 CPU 时间分配（进程调度）、管理内存（虚拟内存）、管理文件（文件系统）、管理设备（驱动程序）……”

为什么这门课重要：

“学完操作系统，你会真正’懂’计算机。你会理解为什么程序有时候会卡、为什么内存不够会崩溃、为什么多线程会出 bug。”

“而且，以后你做后端开发、做分布式系统、做云原生——底层全是操作系统概念的延伸。进程、线程、锁、文件描述符……这些词你以后每天都会遇到。”

给他们的建议：

“操作系统理论课可能有点抽象。建议装个 Linux 自己玩，亲眼看进程、看内存、看文件系统。推荐书：《Operating Systems: Three Easy Pieces》（免费在线版），比国内教材好懂很多。”

计算机网络

一句话： “数据是怎么从你的手机跑到对方手机上的？”

“你发一条微信消息，点’发送’的那一刻，这条消息经历了什么？”

“它先从你的手机出发，经过 Wi-Fi 路由器，进入运营商的网络，经过若干个路由器的转发，到达腾讯的服务器，服务器再通过类似的过程把消息推送到对方手机上。全程不到 0.1 秒。”

“这个过程中，数据被切分成小包，每个包独立传输，在目的地再组装起来。每个环节都有对应的’协议’在规定’怎么发、怎么收、出错了怎么办’。”

“计算机网络就是研究这些协议的：TCP、IP、HTTP、DNS……你们每天都在用，但可能从来没想过它们是什么。”

为什么重要：

“不管你以后做前端、后端、移动端、还是运维，网络知识都是必须的。面试必问、工作必用。而且理解网络之后，你对互联网世界的理解会完全不同。”

给他们的建议：

“重点理解 TCP 三次握手、HTTP 协议、DNS 解析。推荐用 Wireshark 抓包工具，亲眼看你的电脑在上网时到底发了什么数据，非常直观。”

数据库原理

一句话： “几十亿条数据怎么做到秒级查询？”

“你淘宝搜索’运动鞋’，0.2 秒出来几百万条结果。这是怎么做到的？不是一条一条翻——那是大海捞针。靠的是索引。”

“数据库就像一个超级图书馆。数据是书，索引是目录，SQL 是你跟图书管理员说的话（’帮我找 2024 年出版的计算机类书籍’）。管理员怎么高效地找到你要的书——这就是数据库原理在研究的。”

为什么重要：

“几乎所有的软件都有数据要存。用户信息、订单、聊天记录、文章……全部存在数据库里。不懂数据库，你写的程序就只能处理内存里的临时数据，关机就没了。”

给他们的建议：

“SQL 语言一定要熟练——这是跟数据库对话的语言，语法不多，但要写好需要理解底层原理。重点理解索引、事务、范式。”

编译原理

一句话： “你写的代码是怎么变成机器能懂的指令的？”

“你写了一行 int a = 1 + 2;，计算机怎么理解它？编译器会做这几步：先把它拆成一个个词（词法分析），再分析语法结构（语法分析），再检查有没有错误（语义分析），最后生成机器指令（代码生成）。”

“编译原理就是研究这个过程的。学完之后你会理解：为什么有些错误编译器能发现，有些发现不了？为什么有的语言快有的慢？你甚至可以自己写一个小语言。”

给他们的建议：

“编译原理公认是 CS 里最难的课之一。不用害怕——如果你以后不做编译器相关工作，理解原理即可，不需要精通每个细节。但学完之后你对’程序是怎么跑起来的’会有质的理解。”

第四层：方向选择（5 分钟）

“前三层是所有人都要学的。到了第四层，你开始分化，选自己的方向。”

“这些方向你们现在不需要选，大二下或大三再考虑。但我先把路给你摆出来：”

方向 一句话介绍 核心前置知识
人工智能 / 机器学习 让机器从数据中学习规律 数学（线代+概率）+ Python + 深度学习框架
软件工程 把大型软件做好、做稳、做快 设计模式 + 项目经验 + 团队协作
网络安全 攻与防的博弈 网络基础 + 操作系统 + 密码学
计算机图形学 让计算机画出逼真的画面 线性代数 + 图形 API + 物理
嵌入式 / IoT 让硬件变”智能” 组成原理 + 操作系统 + 硬件接口
数据科学 从海量数据中提取价值 概率统计 + 数据库 + 机器学习
分布式系统 / 云计算 让成千上万台机器协同工作 操作系统 + 网络 + 数据库

“看到了吗？每个方向的前置知识，都是你们前三年学的那些课。所以课表里的每一门课，都在为你未来某个方向的选择做准备。”

“不用现在就纠结选什么。大一大二把底层基础打好，大三你就有选择权。 反过来，如果基础没打好，大三想选某个方向可能发现门槛都够不着。”

附：一些额外提醒（5 分钟）

“在讲’后悔故事’之前，插几个课表上没写但很重要的事。”

① 大学物理

“你们可能会想：’我是学计算机的，为什么要学物理？’原因是：物理训练的是建模能力——把一个真实世界的现象变成数学方程。这种能力在计算机里到处用得上。而且物理实验培养的动手习惯也是有用的。”

“不过说实话，大物对 CS 的直接关联没有数学那么大。认真学就好，不用死磕，以前考原题”

② 英语

“英语不是’一门课’，是一个基础设施。”

“你们以后每天都要面对英文：官方文档是英文、Stack Overflow 是英文、GitHub 是英文、论文是英文、最新的技术博客是英文。中文资料大多是二手翻译。”

“不是要你英语多好，但技术英语阅读能力必须有。大一开始就有意识地读一些英文技术文章，慢慢就习惯了。”

③ 不在课表上但很重要的东西

“课表上不会教你 Markdown、Git、命令行、正则表达式、怎么搜资料、怎么提问……这些’隐性技能’。上周已经讲过了，今天不重复。”

“我想强调的是：不要只盯着课表学。 课表给你的是骨架，课表之外的东西是血肉。上周发的《生存手册》里有详细清单，回去看看。”

小结

“这些故事想告诉你们的不是’每门课都要考 90 分’。而是——“

“基础课的价值不是在考试的时候体现的，而是在你以后需要用到它的时候体现的。“

“你大一学的东西，可能大二就忘了具体公式，但那个’理解’会留在你的知识体系里。等到大三做大项目、找实习、做毕设的时候，你会突然发现：’哦，原来这门课是用在这里的。’”

“所以不要为了考试而学，要为了’以后的自己能用上’而学。”

四年节奏概览（5 分钟）

“最后给你们一张时间表，让你们知道每个阶段的重点。”

阶段 时间 重点 关键词
探索期 大一上 认真上课（尤其是数学和C语言），适应大学生活，注册 GitHub，开始接触编程 打基础
扎根期 大一下 学好数据结构和算法，开始刷题，参加社团/技术社区，自学 Python 扎根
深入期 大二 系统层课程（组成原理、OS、网络），开始做项目，找到自己感兴趣的方向 深入
选择期 大三上 选定方向，深入学习，开始实习或科研准备 选择
冲刺期 大三下~大四 实习/考研/求职，做毕设 冲刺

“这张表不是铁律，只是参考。每个人的节奏不同，但大方向差不多。”

“你们现在在第一阶段——探索期。这个阶段最重要的事只有一件：认真上课，打好数学和编程基础。 其他的都不着急。”

Q&A（10 分钟）

预期问题和建议回答：

常见问题 建议回答方向
“大物和政治课也要认真学吗？” “大物对你直接帮助没那么大，但别挂科。政治课……尽量听吧，至少别翘太多。绩点还是有用的，保研、出国、有些公司会看。”
“我该不该加 ACM 队/竞赛？” “ACM 确实锻炼算法能力，对面试很有帮助。但 ACM 不是唯一的路。你可以大一上先了解一下，试几道题，觉得喜欢再深入。不喜欢也不用勉强，做项目也是一条路。”
“数据结构用 C 语言学还是 C++？” “看你们学校怎么教。如果学校用 C 教，就跟学校走。关键是理解数据结构的逻辑，语言只是实现工具。”
“教材用中文还是英文？” “中文教材入门容易，英文教材质量通常更高。建议：入门用中文快速过，然后读一本经典的英文教材。比如操作系统看 OSTEP，算法看 CLRS。”
“需要提前预习吗？” “不强求。如果你有余力，可以提前看看 C 语言的入门视频（B站翁恺的 C 语言课很好）。但数学课不建议提前自学——容易走偏，开学跟着老师走效率更高。”
“考研要什么时候开始准备？” “大三上开始了解，大三下正式准备，完全来得及。大一不需要想这个。”

留作业

“最后两个小任务：”

作业一： 打开你们下载的培养方案，对照今天讲的技能树，把每门课标上它属于哪一层：数学基础 / 编程基础 / 系统层 / 方向课 / 其他。做完之后你们会对整个课程体系有一个非常清晰的认识。”

作业二： 如果你还没有注册 GitHub，今天注册好。如果已经注册了，试着在上面建一个仓库（repository），名字叫 my-cs-notes，把你今天的听课笔记用 Markdown 写进去，push 上去。”

“今天的 PPT 和技能树大图发群里了，建议存下来，以后选课的时候翻出来看看。”
