为什么你应该开始用 Markdown 写作？—— 一篇给新手的友好指南

如果你经常在 GitHub、技术论坛、笔记软件里看到一种用 # * []() 等符号就能排版的神奇文字格式，恭喜你，你遇到了 Markdown。

本文会带你了解：为什么要用 Markdown，以及它最核心的语法。

一、先聊聊：为什么要用 Markdown？

1. 专注内容，而非排版

写 Word 文档时，我们常常会陷入“选什么字体好”、“标题用几号字”、“行间距调多少”的纠结中。Markdown 的理念恰恰相反：用简单的符号标记结构，把排版交给软件。你只需要关心写什么，样式由渲染器统一处理，最终效果反而更整洁一致。

2. 纯文本，跨平台，永远可读

Markdown 文件本质就是一个 .md 或 .markdown 后缀的纯文本文件。用记事本都能打开，不依赖特定软件，不会因为版本不同而乱码。十年后的电脑依然能轻松读取你今天写的笔记。

3. 程序员、写作者的通用语言

  • GitHub、GitLab 上的 README、Issue、Pull Request 都用 Markdown。
  • Notion、Obsidian、Typora、印象笔记 等主流笔记软件原生支持。
  • 许多静态网站生成器（如 Hexo、Hugo、VuePress）基于 Markdown 构建博客。

4. 导出格式极其丰富

写好一份 .md 文件，可以一键导出为：

  • 排版精美的 PDF
  • 结构清晰的 HTML 网页
  • Word 文档（.docx）
  • 甚至 幻灯片（通过 Marp、Slidev 等工具）

5. 版本控制友好

因为 Markdown 是纯文本，用 Git 管理时能清晰看到每一次改动的具体内容（哪一行加了什么字），而 Word 二进制文件则很难比较差异。这对协作写作和知识管理意义重大。

三、几个实用的扩展语法（部分编辑器支持）

下面这些语法在 Typora、Obsidian、GitHub、Notion 等现代工具中普遍可用，能进一步提升写作效率。

四、推荐工具：选一把顺手的“笔”

  1. Typora（经典所见即所得）：编辑时直接显示排版效果，沉浸感强，支持导出多格式。
  2. VS Code + Markdown 插件：程序员首选，一边写代码一边写文档。
  3. Obsidian（双向链接笔记）：知识库管理神器，支持海量插件。
  4. 在线编辑器：如 StackEdit、Dillinger，无需安装。
  5. Notion / 语雀：笔记协作平台，Markdown 语法几乎全兼容。

如果你是新手，推荐先用 Typora 体验纯粹的 Markdown 写作；如果已经有笔记习惯，直接在你常用的软件里新建一个 .md 文件开始练习即可。

五、写在最后

Markdown 不是一门高深的“语言”，它更像一种 让书写回归简洁的习惯。一旦熟悉了那十几个符号，你会发现自己的注意力更容易集中在要表达的思想上，而不是字号和颜色。

现在就可以打开一个空白的 .md 文件，把本文里的语法亲手敲一遍——十分钟后，你就是一个合格的 Markdown 写作者了。

祝你写作愉快！ ✍️
